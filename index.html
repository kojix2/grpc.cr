    grpc

    A native Crystal gRPC library built on libnghttp2.

    Installation

    Add to your shard.yml:

    dependencies:
  grpc:
    github: kojix2/grpc

    Install libnghttp2 (system package):

    # macOS
brew install nghttp2

# Ubuntu / Debian
apt install libnghttp2-dev

# Arch / Manjaro
pacman -S nghttp2

    Quick start

    1. Generate Crystal stubs from a .proto file

    # Build the plugin once
crystal build src/protoc-gen-crystal-grpc.cr -o bin/protoc-gen-crystal-grpc

# Generate stubs
protoc --plugin=protoc-gen-crystal-grpc=bin/protoc-gen-crystal-grpc \
       --crystal-grpc_out=lib/myapp \
       protos/helloworld.proto

    This produces lib/myapp/helloworld.grpc.cr with an abstract service base class and a typed client stub.

    2. Implement the server

    require "grpc"

class GreeterImpl < Helloworld::GreeterService
  def say_hello(req : Helloworld::HelloRequest, ctx : GRPC::ServerContext) : Helloworld::HelloReply
    Helloworld::HelloReply.new(message: "Hello, #{req.name}!")
  end
end

server = GRPC::Server.new
server.handle GreeterImpl.new
server.listen "0.0.0.0", 50051

    3. Call from a client

    require "grpc"

channel = GRPC::Channel.new("localhost:50051")
client  = Helloworld::GreeterClient.new(channel)

reply = client.say_hello(Helloworld::HelloRequest.new(name: "Alice"))
puts reply.message  # => Hello, Alice!

channel.close

    Metadata and deadlines

    # Per-call metadata and deadline
reply = client.say_hello(
  Helloworld::HelloRequest.new(name: "Alice"),
  metadata: {"authorization" => "Bearer token"},
  deadline: 5.seconds,
)

# Or build a reusable context
ctx = GRPC::ClientContext.new(
  metadata: {"x-request-id" => "abc123"},
  deadline: 10.seconds,
)
reply = client.say_hello(req, ctx: ctx)

    Error handling

    begin
  reply = client.say_hello(req)
rescue ex : GRPC::StatusError
  puts ex.code     # GRPC::StatusCode::NOT_FOUND, etc.
  puts ex.message
end

    On the server side, raise GRPC::StatusError to return a specific gRPC status:

    def say_hello(req, ctx)
  raise GRPC::StatusError.new(GRPC::StatusCode::NOT_FOUND, "user not found")
end

    Server context

    def say_hello(req, ctx : GRPC::ServerContext)
  puts ctx.peer              # "127.0.0.1:54321"
  puts ctx.metadata["x-request-id"]?
  puts ctx.deadline          # Time? — nil if no grpc-timeout was sent
  raise GRPC::StatusError.new(GRPC::StatusCode::CANCELLED, "deadline exceeded") if ctx.timed_out?
  # ...
end

    Server streaming

    On the server, override server_streaming? and dispatch_server_stream in your service:

    class MyService < MyPackage::MyService
  def server_streaming?(method : String) : Bool
    method == "ListItems"
  end

  def dispatch_server_stream(method, request_body, ctx, writer)
    case method
    when "ListItems"
      req = MyPackage::ListRequest.from_protobuf(request_body)
      items_for(req).each do |item|
        writer.send(item.to_protobuf)
      end
      GRPC::Status.ok
    else
      GRPC::Status.unimplemented("unknown method")
    end
  end
end

    On the client, use channel.open_server_stream or the generated typed stub:

    stream = channel.open_server_stream("mypackage.MyService", "ListItems", req.to_protobuf)
stream.each do |bytes|
  item = MyPackage::Item.from_protobuf(bytes)
  puts item.name
end
puts stream.status  # OK

    Client streaming

    The client sends multiple request messages; the server returns one response.

    On the server, override client_streaming? and dispatch_client_stream:

    def client_streaming?(method : String) : Bool
  method == "RecordRoute"
end

def dispatch_client_stream(method, requests, ctx, writer)
  # requests : Array(Bytes) — all client messages in order
  total = requests.sum { |b| Point.from_protobuf(b).distance }
  {RouteSummary.new(distance: total).to_protobuf, GRPC::Status.ok}
end

    On the client, pass an array of serialized messages:

    messages = points.map(&.to_protobuf)
body, status = channel.client_stream_call("route.RouteGuide", "RecordRoute", messages)
summary = RouteSummary.from_protobuf(body)

    Bidirectional streaming

    The client sends multiple requests; the server streams multiple responses.

    On the server, override bidi_streaming? and dispatch_bidi_stream:

    def bidi_streaming?(method : String) : Bool
  method == "Chat"
end

def dispatch_bidi_stream(method, requests, ctx, writer)
  requests.each do |b|
    msg = ChatMessage.from_protobuf(b)
    writer.send(ChatMessage.new(text: "echo: #{msg.text}").to_protobuf)
  end
  GRPC::Status.ok
end

    On the client, pass request messages and iterate the response stream:

    messages = chat_messages.map(&.to_protobuf)
stream = channel.open_bidi_stream("chat.Chat", "Chat", messages)
stream.each do |bytes|
  puts ChatMessage.from_protobuf(bytes).text
end

    Note: The current implementation is half-duplex — all client messages are buffered and sent together, after which the server streams its responses. True concurrent send/receive (full bidi) is planned for a future release.

    TLS

    Server

    server = GRPC::Server.new

# From cert/key files
server.use_tls(cert: "/path/to/server.crt", key: "/path/to/server.key")

# Or from a pre-built context
ctx = OpenSSL::SSL::Context::Server.new
ctx.certificate_chain = "/path/to/server.crt"
ctx.private_key = "/path/to/server.key"
server.use_tls(context: ctx)

server.handle MyServiceImpl.new
server.listen "0.0.0.0", 50443

    Client

    # Default TLS — use the system CA bundle (prefix the address with https://)
channel = GRPC::Channel.new("https://myserver.example.com:50443")

# Custom TLS context (e.g. self-signed cert, mutual TLS)
tls_ctx = OpenSSL::SSL::Context::Client.new
tls_ctx.ca_certificates = "/path/to/ca.crt"
tls_ctx.alpn_protocol = "h2"

channel = GRPC::Channel.new("https://myserver.example.com:50443", tls_context: tls_ctx)

    Interceptors

    Interceptors wrap RPC calls on either side of the connection. All four RPC variants (unary, server-streaming, client-streaming, bidi) are supported.

    Server interceptors

    class LoggingServerInterceptor < GRPC::ServerInterceptor
  # Intercept unary calls (required)
  def call(method, request, ctx, next_call)
    STDERR.puts "→ server #{method}"
    result = next_call.call(method, request, ctx)
    STDERR.puts "← server #{result[1].code}"
    result
  end

  # Intercept server-streaming calls (optional — defaults to pass-through)
  def call_server_stream(method, request, ctx, writer, next_call)
    STDERR.puts "→ server_stream #{method}"
    next_call.call(method, request, ctx, writer)
  end

  # call_client_stream and call_bidi_stream follow the same pattern
end

server = GRPC::Server.new
server.intercept LoggingServerInterceptor.new
server.handle MyServiceImpl.new
server.listen "0.0.0.0", 50051

    Client interceptors

    class AuthClientInterceptor < GRPC::ClientInterceptor
  def initialize(@token : String)
  end

  # Intercept unary calls (required)
  def call(method_path, request, ctx, next_call)
    ctx.metadata.set("authorization", "Bearer #{@token}")
    next_call.call(method_path, request, ctx)
  end

  # Intercept server-streaming calls (optional — defaults to pass-through)
  def call_server_stream(method_path, request, ctx, next_call)
    ctx.metadata.set("authorization", "Bearer #{@token}")
    next_call.call(method_path, request, ctx)
  end

  # call_client_stream and call_bidi_stream follow the same pattern
end

channel = GRPC::Channel.new(
  "localhost:50051",
  interceptors: [AuthClientInterceptor.new("my-secret")] of GRPC::ClientInterceptor,
)

    The first interceptor in the array executes outermost (first on the way in, last on the way out).

    Message compression

    gRPC messages can be gzip-compressed at the codec level. This is transparent on the receiving end — both compressed and uncompressed frames are decoded automatically.

    # Encode a message with gzip compression
framed = GRPC::Codec.encode(message_bytes, compress: true)

# Decode handles both compressed and uncompressed frames transparently
decoded, consumed = GRPC::Codec.decode(framed)

    The server and channel use uncompressed frames by default. Per-message compression can be enabled at the codec layer when building custom transports.

    Running the examples

    Helloworld (unary)

    # Terminal 1 — server
crystal run examples/helloworld/server.cr

# Terminal 2 — client
crystal run examples/helloworld/client.cr -- localhost 50051 Alice
# => Sending: SayHello(name=Alice)
# => Response: Hello, Alice!

    Streaming (all four RPC styles)

    # Terminal 1 — server
crystal run examples/streaming/server.cr

# Terminal 2 — client
crystal run examples/streaming/client.cr
# => Square(7)    = 49
# => Range(5)     = 1 2 3 4 5
# => Sum(1..5)    = 15
# => Transform([2,3,4]²) = 4 9 16
